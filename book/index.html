Building Agent Sandboxes

Learn how to safely execute untrusted code in isolated VM-level environments using AWS Lambda MicroVMs and Firecracker. This course walks you through building production-ready agent sandboxes for AI coding assistants, security testing, and multi-tenant applications—with runnable Python examples, failing-test challenges, and inline citations to official AWS and Firecracker documentation.

Click Launch Codespace in the sidebar for a ready-to-go environment with Python and the AWS CLI already configured, or clone the repo and run pip install -e . locally.

Part I · Foundations

  1. 1Why Agent Sandboxes MatterThe security problem of running untrusted code, what can go wrong, and when VM-level isolation is the right choice.
  2. 2Understanding Firecracker MicroVMsHow Firecracker differs from Docker and traditional VMs, the minimal five-device model, and <125ms startup.
  3. 3Lambda MicroVMs ArchitectureImage-based deployment, suspend/resume lifecycle, state persistence, and authentication with JWE tokens.

Part II · Getting Started with Lambda MicroVMs

  1. 4Your First MicroVM ImageWriting a Dockerfile, creating a Flask app, and using CreateMicrovmImage to build and snapshot.
  2. 5Launching and ConnectingRunMicrovm with idle policies, authentication tokens, and making your first request.
  3. 6Lifecycle ManagementManual suspend/resume, auto-suspension for cost optimization, and graceful termination.

Part III · Building an Agent Sandbox

  1. 7Executing Untrusted CodeDesigning a code execution API, capturing output, timeout handling, and security boundaries.
  2. 8Package Management and DependenciesRuntime vs. image-time installation, using pip inside the MicroVM, and caching strategies.
  3. 9File System IsolationUnderstanding persistent storage, cleaning up user files, quota management, and disk limits.
  4. 10Resource Limits and QuotasCPU/memory configuration, preventing resource exhaustion, and the 8-hour maximum runtime.

Part IV · Advanced Patterns

  1. 11Multi-Tenant IsolationOne MicroVM per user vs. shared instances, session management, and cost/security tradeoffs.
  2. 12Security HardeningNetwork restrictions, validating user code, monitoring malicious behavior, and Firecracker's layers.
  3. 13Observability and DebuggingCloudWatch integration, tracing execution, debugging failures, and cost monitoring.
  4. 14Production PatternsImage versioning, blue/green deployment, scaling decisions, and designing for the 8-hour limit.

Part V · Real-World Agent Systems

  1. 15AI Coding Assistant PatternHow Claude Code and Cursor use sandboxes, REPL in a MicroVM, and integrating with LLM tool calling.
  2. 16Security Testing SandboxRunning vulnerability scanners, malware analysis, network isolation, and forensic snapshots.
